Recently I've been thinking about choppy animated faces. The kind that uses photographs of real people at a super low frame-rate, clearly in an attempt to look realistic at a time when tech couldn't achieve it.

Think FMV's, 90's digital rotoscoping and The Last Express.

There's something either creepy or charming about this technique whenever I see it. The result is always weirdly nice or horrifying. I've been thinking about how this can be applied to the web and now that's what I'm subjecting you to.

The goal being to create a face that responds to mouse movements.

The Imagery

I found this set of headshots from some scientific study that I liked. There's a variety of emotions with not a lot of in-between frames so animating the shots together would get the choppy effect I'm going for.

Dumped the image into Aseprite, sliced up the frames and added flipped copies of the left facing images to create a 180 degree turn.

Here's the isolated animations for each expression.

Now I have a spritesheet containing all frames for the 180 degree turns for each expression.

Putting it on the Web

My idea was to display the image as a single frame from a spritesheet using the CSS background-image and background-position properties, then overlay a grid which will change the background-position to display a different frame from the spritesheet depending on which cell from the grid was hovered.

This is what you see below. The top row is the angry face images, middle row is neutral and the happy faces are on the bottom row. Her face turns relative to the column you're hovering.

It's works exactly how I pictured it. She's horrific.

This was hacked together just for this page so all the background-position's of the overlay cells were manually set, along with the number of cells to display proportionate to the number of frames and their positions.

If I were to actually use this for something I'd write a sass mixin to spit out the right number of nth-child styles based on the number of frames and the width of the frames in that spritesheet.

Further Usage

A better use for this (what I actually had in mind before I found that creepy woman) is to have the face follow your cursor.

I may actually end up doing something with this if I can think of a cool enough application. I know there's other, more modern, fully CSS solutions to transform the direction of images and even 3D models to follow the cursor - but something about the choppiness of these real photographs really appeals to me.

We'll see what I come up with.
